Activity 1 - IoT Apps

What is an IoT Application?

  • Applications that interconnects devices so they can communicate with each other without any human interference

Examples:

  • Smart building
  • Traffic monitoring system
  • Security system
  • Healthcare monitoring
  • Inventory management
  • Driveless car
  • Military drone

Main features:

  • Data management (with Cloud Computing)
  • Scalability
  • Speed
  • Security
  • Intelligent
  • Concurrency (for Complex Systems)
  • Communication - different kind of networks
  • Sensor and Device
  • Applications - toolbox, software, framework

Activity 2 - Smart Building

What does smart mean?

  • Able to integrate with other devices
  • Use data to interact with you

Some smart building scenarios:

  • Lightning management
  • Security system
  • Hazard detection
  • Monitor power/energy useage
  • Monitoring temperature and scheduling the AC running time
  • Kitchen plans and menu

Main features:

  • Sensor technology, intelligent technology, RDFI - radio frequency identification
  • Cloud computing
  • Storage, collection of data
  • Regulate the environment - temperature, humidity and air condition
  • Centralised event processing: execute function to monitor, navigate and optimize process in real-time
  • Wirless communication - Wifi, Bluetooth
  • Battery/Power source

Activity 3 - Control Interface

High level features:

  • The user may have several smart buildings, so the interface should show the name for the building it is currently control
  • It needs a display to show a status of the sensors, such as which light are connected to the system and whether they are on or off, or the room currently temperature
  • It needs a mechanism to add, edit and remove component to the system, such as adding additional light
  • It needs to take the userâ€™s input and turn it into commands for the system. This could be as simply as clicking on a button to directly toggle a light, or complex such as providing a schedule for the desire temperature to be maintained at different time of the day.
  • The interface needs an authorization and authentication mechanism to ensure that control of the system is limited to the permitted users
  • There should be a mechanism for auditing the device and pushing firmware updates to the components
